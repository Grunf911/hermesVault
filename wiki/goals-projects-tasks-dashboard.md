# Goals, Projects & Tasks Dashboard

## Core model
- Goals = long-term vision and outcomes.
- Projects = mid-term initiatives that take multiple tasks.
- Tasks = atomic actions, one-offs, or steps inside a bigger project or goal.

## Decision tree
1. Is this a long-term desired outcome, direction, or vision?
   - Yes -> create or update a Goal.
2. Is this a multi-step initiative that moves a goal forward?
   - Yes -> create or update a Project.
3. Is this a single next action, a quick brain-dump item, or a one-off task?
   - Yes -> create or update a Task.
4. Is it strategic enough to matter, but still too broad to execute today?
   - Put it in a Goal first, then break it into Projects later.
5. Does it need to stay out of the current focus lane for now?
   - Park it as a non-focused task/project and leave it outside focus mode until needed.

## Focus mode
- The Goal-level SetFocus option turns on focus for that Goal.
- When focus is active, filtering Projects and Tasks by focus only shows items directly tied to the focused Goal.
- Use focus mode to remove overwhelm while still allowing unrestricted brain-dump capture.
- This means you can add future work without cluttering the active lane.

## Practical rules
- Use Goals for direction.
- Use Projects for initiatives.
- Use Tasks for execution.
- If the work is actionable today, it should usually be a Task.
- If the work needs coordination or several steps, it should usually be a Project.
- If the work is mainly about where you want to go, it should be a Goal.

## Relationships
- Goals can point to Projects.
- Projects can point to Tasks.
- Tasks can also point directly to Goals when they are not part of a project.
- Direct Goal-linked tasks are useful for quick capture and brain-dump cleanup.
